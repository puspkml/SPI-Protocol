# SPI Master-Slave Communication System in Verilog

![Verilog](https://img.shields.io/badge/Language-Verilog-blue)
![Simulation](https://img.shields.io/badge/Simulation-Icarus%20Verilog-green)



## Overview

This project implements a configurable **SPI (Serial Peripheral Interface) communication system** in Verilog, consisting of a synthesizable SPI Master, SPI Slave, and a complete testbench environment.

The design supports both **read and write transactions**, configurable SPI modes through **CPOL** and **CPHA**, simulation using **Icarus Verilog**, waveform analysis using **GTKWave**, and synthesis using **Yosys**.

The project demonstrates end-to-end SPI communication where:

* The master initiates transactions.
* Address and data are transferred serially through MOSI.
* Read data is returned through MISO.
* The slave decodes commands and responds accordingly.
* Simulation results can be observed in generated VCD waveforms.

## SPI Frame Format

### Write Transaction

```text
+---------+------------+---------+
| RW = 1  | Address    | Data    |
| 1 bit   | 8 bits     | 8 bits  |
+---------+------------+---------+
```

### Read Transaction

```text
+---------+------------+
| RW = 0  | Address    |
| 1 bit   | 8 bits     |
+---------+------------+

Slave returns 8-bit data on MISO.
```

## Getting Started

### Prerequisites

Install the following tools:

* Icarus Verilog
* GTKWave
* Yosys

Ubuntu:

```bash
sudo apt update
sudo apt install iverilog gtkwave yosys
```

---

## Running Simulation

Compile the design:

```bash
iverilog -o sim spi_master.v spi_slave.v tb_system.v
```

Run simulation:

```bash
vvp sim
```

Expected console output:

```text
WRITE RESULT
Slave Address = 12
Slave Data    = AB

READ RESULT
Master Data = 5A
```

## Viewing Waveforms

Open the generated VCD file:

```bash
gtkwave spi_system.vcd
```

Useful signals to observe:

```text
clk
sclk
cs_n
mosi
miso
tx_en
rx_en
slave_address
slave_data
data_in
rx_valid
```

These signals provide a complete view of SPI transaction timing and protocol behavior.

## Example Transaction Flow

### Write Operation

Master sends:

```text
RW      = 1
Address = 0x12
Data    = 0xAB
```

Slave stores:

```text
Address = 0x12
Data    = 0xAB
```

### Read Operation

Master sends:

```text
RW      = 0
Address = 0x34
```

Slave responds:

```text
0x5A
```

Master receives:

```text
data_in = 0x5A
rx_valid = 1
```

## How It Works

### SPI Master

Responsibilities:

* Generates SPI clock
* Controls chip select
* Sends read/write command bit
* Transmits address and data
* Receives incoming data during read operations

### SPI Slave

Responsibilities:

* Detects read/write requests
* Captures address information
* Stores incoming write data
* Drives MISO during read transactions
* Supports configurable SPI modes
